Short Answer Type Questions
1. Explain the terms View. [WBUT 2006, 2008, 2012]
Answer:
A view is a relation (virtual rather than base) and can be used in query expressions, that
is, queries can be written-using the view as a relation. In other words, a view is a named
table that is represented, not by its own physically separate stored data. but by its
definition in terms of other named tables (base tables or views). The base relations on
which a view is based are sometimes called the existing relations. The definition of a
view in a create view statement is stored in the system catalog. The syntax to create a
view is: ‘ A
CREATE [OR REPLACE] VIEW <view_name> [(<aliases>) ] As
<query> WITH {READ ONLY|CHECK OPTION [CONSTRAINT
<constraint_name>] }
