In SQL Server we can create four types of triggers Data Definition Language (DDL)
triggers, Data Manipulation Language (DML) triggers, CLR triggers and Logon triggers.
1. DDL Triggers '
In SQL Server we can create triggers on DDL, statements (like CREATE, ALTER, and
DROP) and certain system defined stored procedures that perform DDL-like operations.
Example: If you are going to execute the CREATE LOGIN statement or the sp_addlogin
stored procedure to create login user, then both these can execute/fire a DDL trigger that
you can create on CREATE_LOGIN event of SQL Server.
We can use only FOR/AFTER clause in DDL triggers not INSTEAD OF clause means
we can make only After Trigger on DDL statements.
DDL trigger can be used to observe and control actions performed on the server, and to
audit these operations. DDL triggers can be used to manage administrator tasks such as
auditing and regulating database operations.
2. DML Triggers |
‘In SQL Server we can create triggers on DML statements (like INSERT. UPDATE, and
DELETE) and stored procedures that perform DML-like operations. DML Triggers are of
