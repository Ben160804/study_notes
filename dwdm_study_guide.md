**Must-know definitions:**
- Data Warehouse: Subject-oriented, integrated, time-varying, non-volatile (Inmon's 4 characteristics)
- Data Mart: Department-specific subset of DW

- Metadata: Data about data (technical, business, operational)

**APRIORI ALGORITHM (L861-L890 — FULL ANSWER IN ORGANIZER):**
```
Key Concepts:
- Frequent Itemsets: sets of items with minimum support (k-itemset)
- Apriori Property: Any subset of frequent itemset must be frequent
- Join Step: Cₖ generated by joining Lₖ₋₁ with itself
- Prune Step: Any (k-1)-itemset not frequent → cannot be subset of frequent k-itemset

Pseudo-code:
L₁ = {frequent items}
For (k = 2; Lₖ₋₁ ≠ ∅; k++) do begin
    Cₖ = candidates generated from Lₖ₋₁
    for each transaction t in database do
        increment count of all candidates in Cₖ contained in t
    Lₖ = candidates in Cₖ with min_support
end
return ∪ Lₖ
```

**FP-GROWTH (L1524-L1527 — RECURRING 2010, 2012, 2013, 2018):**
- No candidate generation (unlike Apriori)
- Build FP-tree → Mine conditional FP-trees recursively
- Phases: (1) Build header table & FP-tree, (2) Mine conditional pattern bases, (3) Construct conditional FP-trees, (4) Recurse

**DECISION TREE INDUCTION (L1326, L2085):**
- ID3: Entropy / Information Gain
- C4.5: Gain Ratio (handles multi-value attributes)
- CART: Gini Index, binary splits
- CLS Algorithm (2017): Classifier based on clustering

**Must-know formulas:**
- Entropy: H(S) = -Σ pᵢ log₂ pᵢ
- Info Gain: IG(S,A) = H(S) - Σ(|Sᵥ|/|S|) H(Sᵥ)
- Gain Ratio: GR = IG / SplitInfo
- Gini: 1 - Σ pᵢ²
- Support(A→B) = P(A∪B)
- Confidence(A→B) = P(B|A) = Support(A∪B)/Support(A)
- Lift = Confidence / P(B)

**Gap ⚠️ Clustering Detail** — Syllabus: K-means, hierarchical, partitioning, density-based (DBSCAN). Only one PYQ (L922). Need textbook (Han & Kamber Ch. 10).
